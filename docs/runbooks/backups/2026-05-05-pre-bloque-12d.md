**Smoke UI** (5 may 14:25 - 6 may 04:24):

1. **Bug 1 detectado**: form Filament Page `ImportarSgip` reportaba "Sube
   un CSV primero" tras upload visual exitoso. Causa raíz: `$this->data['csv']`
   queda NULL hasta que Livewire sync. **Fix aplicado**: usar
   `$this->form->getState()` para forzar sync antes de leer el path.

2. **Bug 2 detectado**: Service `parseCsv` lanzaba `CSV missing columns:
   Asignada a` con CSV real. Causa raíz: el CSV exportado de SGIP tiene
   line endings mezclados (CRLF + CR-only Mac Classic + LF). PHP `fgetcsv`
   default no detecta CR-only → leía las 28 filas como 1 sola gigante con
   175 columnas. **Fix aplicado**: pre-procesamiento en `parseCsv` que
   normaliza line endings (`\r\n` y `\r` → `\n`) usando `php://temp`
   stream antes de `fgetcsv`.

3. **Bug 3 detectado** durante import: SQLSTATE[22001] Data too long for
   column `panel_id_sgip` at row 1. Causa raíz: el CSV real tiene formato
   `PANEL <id> <NOMBRE_LOCALIDAD>` con longitud hasta 39 chars (ej "PANEL
   17474B SAN SEBASTIAN DE LOS REYES"), no `PANEL XXXXX` 5 dígitos como
   asumía mi prompt. **Fix aplicado**: migration ALTER
   `2026_05_06_120000_extend_lv_averia_icca_panel_id_sgip_length` que
   amplia varchar(20) → varchar(100). Aplicada en prod (53ms).

4. **Bug 4 detectado**: `archivo_origen` guardaba ULID interno de Filament
   FileUpload (`01KQXSDNTG3MVTFM5GEPYJF43Y.csv`) en lugar del filename
   original, perdiendo el audit trail útil. **Fix aplicado**:
   `preserveFilenames(true)` en FileUpload del page.

5. **Bug 5 UX detectado**: columna acciones "Ver" aparecía como pegote
   visual flotante (sticky overlay default Filament 3 con sombra y fondo
   gris desconectado). **Fix aplicado**: `ActionsPosition::AfterColumns`
   integra la columna como columna real fluida + ViewAction con
   label "Ver" + size Small + color gray (estilo ghost). Bonus
   `recordAction('view')` para click row también abre slideOver.
